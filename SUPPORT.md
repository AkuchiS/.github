# Getting help

Happy to help — here's the fastest path to an answer.

1. **Read the project's README.** Most "how do I…" questions are answered there,
   usually near the top.
2. **Search existing issues.** Someone may have hit the same thing already.
3. **Open an issue.** If it's a bug, use the bug report template and include the
   command you ran, your OS, and any error output. If it's a question, just ask —
   no template needed.
4. **Discussions**, where enabled, are the place for ideas, show-and-tell, and
   open-ended questions that aren't bug reports.

## A fair expectation

These are free, open-source tools built and maintained by one person in their own
time. Support is best-effort and given gladly — but it isn't a paid SLA, so please
be patient.

If you rely on a tool commercially and need guaranteed turnaround, the paid tiers
(e.g. **FlowProof Pro** at <https://fp.akuchis.com>) come with priority support and
a direct line.
