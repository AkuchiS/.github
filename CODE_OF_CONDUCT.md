# Code of conduct

A short one, because it isn't complicated.

## The standard

Be decent. Assume good faith. Critique code and ideas, not people. Help newcomers
the way you'd have liked to be helped. Disagreement is fine and useful — contempt
isn't.

## Not welcome

Harassment, personal attacks, discrimination, sustained disruption, or sharing
someone's private information. Doesn't matter how good your patch is.

## Scope

This applies in all project spaces — issues, pull requests, discussions — and in
public spaces when you're representing the project.

## Enforcement

If someone's out of line, report it to the maintainer privately via the
repository's **Security → Report a vulnerability** channel (it's the simplest
private route) or by flagging the comment. Reports are handled discreetly.
Consequences range from a warning to a permanent ban, depending on severity.
The maintainer's call is final.

This is adapted in spirit from the [Contributor Covenant](https://www.contributor-covenant.org/),
trimmed to what a small project actually needs.
